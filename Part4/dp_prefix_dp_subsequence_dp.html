Prefix DP vs Subsequence DP
How to quickly recognise the pattern and write the correct transition.

1. Prefix DP (1D) “No comparisons” family

State is based on a prefix index i. We never compare values like a[i] with a[j]. We only ask: “What can we do using the first i positions?”

Typical state & transition

  • State: dp[i] — best answer for prefix [0..i-1].
  • Transition: combine earlier prefixes:
    dp[i] = F( dp[0], dp[1], ..., dp[i-1] )
  • No a[i] vs a[j] conditions.

In all of these, we only care about “how far” (prefix length), not which element we pick as part of a subsequence.

2. Subsequence DP Comparison-based Prefix DP

State still uses indices, but transitions depend on comparing elements or characters. We decide whether to use or skip something.

2.1 1D Subsequence DP

  • State: dp[i] — best subsequence ending at index i.
  • Transition: look at all j < i and compare:
    if a[j] < a[i]: dp[i] = max(dp[i], dp[j] + 1)

2.2 2D Subsequence DP

  • State: dp[i][j] — best relation between prefixes S[0..i-1] and T[0..j-1].
  • Transition (example, subsequence counting):
    If S[i-1] == T[j-1]:
    dp[i][j] = dp[i-1][j-1] + dp[i-1][j]
    else:
    dp[i][j] = dp[i-1][j]

Key feature: transitions only valid when a comparison condition holds (e.g. <, >, ==).

How to Tell: Prefix DP or Subsequence DP?

Don’t look at “one string vs two strings”. Look at the transition pattern and the problem wording.

Prefix DP (1D)
  • No comparisons like a[i] < a[j] or S[i] == T[j].
  • dp[i] depends only on earlier prefixes:
    dp[i-1], dp[i-2], ..., dp[0].
  • Often linear or “cut at j” style:
    dp[i] = combine(dp[0..i-1])

Think: “How many ways / min cost / max value up to position i?”

Subsequence DP
  • There is ALWAYS some comparison condition.
  • 1D: compare a[i] with previous elements a[j].
  • 2D: compare characters S[i-1] and T[j-1].
  • Transition has “use or skip” flavour:
    use current vs skip current.

Think: “Should I take this element/character into the subsequence or ignore it?”

One-sentence rule: If your DP transition never compares values or characters, it’s Prefix DP. As soon as you see conditions like a[j] < a[i] or S[i-1] == T[j-1], you are in Subsequence DP.

Why Students Won’t Mistake Subsequence DP for Prefix DP

Because the problem wording and the transition pattern look very different.

Typical wording of Prefix DP (1D) problems

  • “Minimum cost to reach step i …”
  • “Can we segment the prefix into words …”
  • “Max profit up to day i …”
  • “Number of ways to reach index i …”
  • No mention of “increasing / decreasing / matching / aligned / common subsequence”.

Typical wording of Subsequence DP problems

  • “Longest increasing / decreasing subsequence …”
  • “Maximum sum increasing subsequence …”
  • “Longest common subsequence of two strings …”
  • “Minimum edits (insert/delete/replace) to turn A into B …”
  • “How many subsequences of S equal T?”

As soon as the problem talks about increasing / decreasing / matching / common / subsequence / convert one string to another, students should immediately think: Subsequence DP (comparison-based).

Teaching tip: Train students to ask this first question:

“Does the DP need to compare positions or characters?”
• If no → start from Prefix DP templates.
• If yes → start from Subsequence DP templates (1D or 2D).
